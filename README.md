# Serverous

## Запуск

```
docker compose up -d
```

## Тестирование API:

Необходимо перейти на

`http://localhost:8080/reference`

## Авторизация для сессий

>Для каждого запроса к `/sessions/` обязательно передавайте Header `UserId`
>Это необходимо для авторизации пользователя (упрощённый вход)

При запуске проекта автоматически создаются 2 пользователя с UserId:
```11111111-1111-1111-1111-111111111111``` и ```22222222-2222-2222-2222-222222222222```

## Фильтры серверов

- OperatingSystem - это Enum, имеет следующие значения:
```
Ubuntu = 0,
Debian = 1,
ArchLinux = 2,
CentOS = 3,
Fedora = 4
```

При фильтрации это поле можно не заполнять, по-умолчанию будут выбраны все доступные ОС

Для выбора конкретной ОС необходимо указать числовое значение параметра operatingSystem

Если используете фильтры, обязательно указывайте максимальные значения (`MaxRamGiB`, `MaxStorageGiB`, `MaxCpuCoreCount`), иначе список серверов будет пустым

## Описание статусов сессии

- `Starting` - сессия активирована, происходит запуск сервера
- `InUse` - сессия используется, запущен таймер
- `Stopped` - сессия прервана пользователем
- `Expired` - время сессии вышло (20 минут)
- `Failed` - произошла ошибка во время сессии

Session.IsActive - это индикатор актуальности сессии. Если `IsActive = true`, сервер, привязанный к сессии, забронирован (запускается или запущен)
